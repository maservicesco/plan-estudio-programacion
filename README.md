# 🚀 Plan de Estudio — De PM a Desarrollador Full-Stack (con puente a Ciberseguridad)

> **Alumno:** perfil Product Manager con experiencia, aprendiendo a programar desde la base.
> **Profesor:** Claude.
> **Objetivo:** nivel *junior empleable* con portafolio, lo antes posible.
> **Ritmo:** intensivo (~10 h/día, ~50-70 h/semana). Estimado: **3-4 meses** a nivel junior.
> **Inicio:** 2026-07-01.

---

## 🎯 La ruta y el porqué

**Ruta principal:** Full-Stack **JavaScript / TypeScript** → **Python** como segundo idioma → puente a **Ciberseguridad**.

Por qué esta ruta y no otra:
1. **Máxima empleabilidad junior** — frontend, full-stack y Node backend concentran la mayor cantidad de vacantes de entrada.
2. **La web es la base #1 de la ciberseguridad** — para AppSec / pentesting web necesitas entender HTTP, sesiones, auth, bases de datos e inyecciones. Construir full-stack **es** cargar esa base.
3. **Bases transversales de seguridad desde el día 1** — Linux, redes y OWASP Top 10 integrados, para que el salto a ciberseguridad sea corto.
4. **Ventaja PM** — producto, usuarios, priorización y comunicación son diferenciadores en entrevistas junior. Los explotamos en el portafolio.

---

## 🧭 Cómo trabajamos juntos (metodología)

- **Ciclo diario:** teoría corta → práctica guiada → ejercicio solo → repaso/feedback.
- **Regla 20/80:** ~20% teoría, ~80% escribir código con mis manos.
- **"Explícamelo tú":** cierras cada tema explicándomelo con tus palabras (técnica Feynman). Si no lo puedes explicar, no lo dominás aún.
- **Sin copy-paste ciego:** puedes usar IA como guía, pero cada línea la entiendes. Yo te voy a preguntar "¿por qué esta línea?".
- **Proyectos reales:** cada fase termina en algo que va al portafolio.
- **Tracking:** registramos avance en [`PROGRESO.md`](PROGRESO.md).

## 📚 Detalle de cada fase

### Fase 0 — Fundamentos, setup y evaluación
- Cómo funciona una computadora, internet y el modelo **cliente-servidor** (mapa mental).
- **Terminal / línea de comandos** (también base de ciberseguridad).
- **Git y GitHub** (control de versiones — se usa TODO el tiempo).
- Instalación: VS Code, Node.js, Git.
- **Evaluación inicial** para medir tu nivel real de partida.

### Fase 1 — Lógica y fundamentos (JavaScript)
- Variables, tipos de datos, operadores.
- Condicionales (`if/else`, `switch`).
- Bucles (`for`, `while`).
- Funciones (parámetros, retorno, scope).
- Estructuras de datos: **arrays** y **objetos**.
- Métodos clave: `map`, `filter`, `reduce`, `find`.
- Manejo de errores y **debugging**.
- Pensamiento algorítmico / resolución de problemas.

### Fase 2 — Frontend
- **HTML** semántico.
- **CSS**: box model, Flexbox, Grid, responsive design.
- **JS en el navegador**: DOM, eventos.
- **Asíncrono**: `promises`, `async/await`, `fetch` a APIs.

### Fase 3 — TypeScript + React
- **TypeScript**: tipos, interfaces, por qué el tipado te salva.
- **React**: componentes, props, `useState`, `useEffect`, listas, formularios, routing.
- Consumo de APIs desde React.

### Fase 4 — Backend + Bases de datos
- **Node.js + Express**: servidor, rutas, middleware.
- **APIs REST**: métodos HTTP, códigos de estado, JSON.
- **SQL + PostgreSQL**: modelado, CRUD, relaciones, JOINs.
- **Autenticación**: hashing de contraseñas, JWT, sesiones *(puente directo a seguridad)*.

### Fase 5 — Full-Stack + Deploy (Capstone)
- Conectar frontend + backend + base de datos en un proyecto real.
- Variables de entorno, buenas prácticas.
- **Deploy** a internet (Vercel / Render).
- Testing básico.

### Fase 6 — Puente a Ciberseguridad
- **Linux**: navegación, permisos, procesos.
- **Redes**: HTTP/HTTPS, DNS, TCP/IP, puertos.
- **Seguridad web — OWASP Top 10**: inyección SQL, XSS, CSRF, auth rota.
- Auditar y endurecer tu propia app capstone.

### Transversal (todo el tiempo)
- **Git/GitHub** en cada proyecto.
- **Inglés técnico** (leer docs, errores).
- **Portafolio** en GitHub + web personal.
- **Python** como segundo idioma (arranca en paralelo desde Fase 3-4).
- **Preparación laboral**: CV, LinkedIn, perfil GitHub, entrevistas técnicas.
